Items in Mythic Framework are defined with properties that control their behavior, appearance, and usage. Understanding the item structure is essential for creating custom items and integrating with the inventory system.

## Item Definition Structure

Items are defined in `shared/items.lua` files across resources:

```lua
{
    name = "water",                      -- Unique identifier
    label = "Water Bottle",              -- Display name
    description = "A refreshing bottle of water",
    weight = 0.5,                        -- Weight in kg
    type = "consumable",                 -- Item type
    image = "water.png",                 -- Image filename (in ui/assets/items/)

    -- Usage properties
    usable = true,                       -- Can be used from inventory
    unique = false,                      -- Each instance is unique
    stackable = true,                    -- Can stack in slots

    -- Quantity limits
    maxStack = 10,                       -- Max stack size (if stackable)

    -- Metadata requirements
    metadata = {},                       -- Default metadata

    -- Decay/durability
    decay = false,                       -- Item decays over time
    decayRate = 0,                       -- Decay per minute

    -- Special properties
    illegal = false,                     -- Illegal item (police can search)
    sellable = true,                     -- Can be sold to shops
    price = 10,                          -- Base price
    blackmarket = false,                 -- Only sold on black market

    -- Equipment properties (for wearables)
    equipSlot = nil,                     -- "head", "body", "accessory", etc.

    -- Weapon properties (for weapons)
    weapon = false,                      -- Is this a weapon
    ammoType = nil,                      -- Ammo type if weapon
    throwable = false                    -- Can be thrown
}
```

## Required Fields

<ParamField path="name" type="string" required>
  Unique item identifier

  **Rules:**
  - Lowercase, no spaces
  - Use underscores for multi-word items
  - Descriptive and consistent

  **Examples:** `"water"`, `"weapon_pistol"`, `"lockpick"`, `"driver_license"`
</ParamField>

<ParamField path="label" type="string" required>
  Display name shown to players

  **Examples:** `"Water Bottle"`, `"Pistol"`, `"Lockpick"`, `"Driver's License"`
</ParamField>

<ParamField path="weight" type="number" required>
  Item weight in kilograms

  **Guidelines:**
  - Small items: 0.1 - 0.5 kg
  - Medium items: 0.5 - 2 kg
  - Large items: 2 - 10 kg
  - Very heavy: 10+ kg
</ParamField>

## Optional Fields

### Display Properties

<ParamField path="description" type="string" optional>
  Detailed item description shown in tooltips

  **Example:**
  ```lua
  description = "A refreshing bottle of water. Restores thirst when consumed."
  ```
</ParamField>

<ParamField path="image" type="string" optional default="default.png">
  Image filename in `ui/assets/items/` directory

  **Format:** PNG, recommended 150x150px

  **Example:** `"water.png"`, `"weapon_pistol.png"`
</ParamField>

### Usage Properties

<ParamField path="usable" type="boolean" optional default="false">
  Whether the item can be used from inventory

  **Example:**
  ```lua
  usable = true  -- For consumables, tools, etc.
  usable = false -- For passive items, materials
  ```
</ParamField>

<ParamField path="unique" type="boolean" optional default="false">
  Whether each item instance is unique (can't stack, has unique metadata)

  **Use For:**
  - Weapons (serial numbers)
  - Phones (phone numbers)
  - ID cards (character-specific)
  - Vehicles keys (specific vehicle)

  **Example:**
  ```lua
  unique = true  -- Each weapon has unique serial, ammo, durability
  ```
</ParamField>

<ParamField path="stackable" type="boolean" optional default="true">
  Whether items can stack in a single slot

  **Example:**
  ```lua
  stackable = true   -- Water bottles can stack
  stackable = false  -- Weapons don't stack
  ```
</ParamField>

<ParamField path="maxStack" type="number" optional default="99">
  Maximum stack size if `stackable = true`

  **Example:**
  ```lua
  maxStack = 50  -- Max 50 water bottles per slot
  ```
</ParamField>

## Item Types

The `type` field categorizes items:

```lua
type = "consumable"   -- Food, drinks, medical items
type = "weapon"       -- Firearms, melee weapons
type = "ammo"         -- Ammunition
type = "tool"         -- Lockpicks, repair kits, fishing rods
type = "material"     -- Crafting materials, resources
type = "valuable"     -- Jewelry, collectibles
type = "document"     -- ID cards, licenses, contracts
type = "clothing"     -- Wearable items
type = "key"          -- Vehicle keys, building keys
type = "container"    -- Backpacks, bags
type = "misc"         -- Miscellaneous items
```

## Item Metadata

Metadata stores instance-specific data for unique or stackable items:

### Common Metadata Fields

```lua
metadata = {
    -- Weapons
    serial = "ABC123456",
    durability = 95,
    ammo = 10,
    attachments = { "flashlight", "suppressor" },

    -- Consumables
    quality = "excellent",
    expiration = 1705334400,  -- Unix timestamp

    -- Documents
    name = "John Doe",
    dob = "1990-01-15",
    photo = "https://...",
    issued = 1700000000,
    expires = 1800000000,

    -- Vehicles
    plate = "ABC123",
    vin = "1HGBH41JXMN109186",
    fuel = 75,
    mileage = 12500,

    -- Containers
    items = {
        { name = "water", count = 5 },
        { name = "sandwich", count = 3 }
    },

    -- Custom data
    customField = "value",
    owner = 123,
    craftedBy = 456,
    craftedAt = 1705334400
}
```

### Accessing Metadata

```lua
-- Get item with metadata
local item = COMPONENTS.Inventory:GetItemInSlot(char.SID, 1)

if item then
    print('Item:', item.label)
    print('Serial:', item.metadata.serial)
    print('Durability:', item.metadata.durability)
    print('Ammo:', item.metadata.ammo)
end

-- Add item with metadata
COMPONENTS.Inventory:AddItem(char.SID, 'weapon_pistol', 1, {
    serial = GenerateSerial(),
    durability = 100,
    ammo = 12,
    attachments = {}
})

-- Update metadata
local item = COMPONENTS.Inventory:GetItemInSlot(char.SID, 1)
item.metadata.ammo = 10
item.metadata.durability = 95

COMPONENTS.Inventory:UpdateSlot(char.SID, 1, item)
```

## Creating Custom Items

### Step 1: Define the Item

```lua
-- shared/items.lua (in your resource)

Config.Items = {
    ["energy_drink"] = {
        name = "energy_drink",
        label = "Energy Drink",
        description = "Restores energy and gives a temporary speed boost.",
        weight = 0.3,
        type = "consumable",
        image = "energy_drink.png",

        usable = true,
        unique = false,
        stackable = true,
        maxStack = 5,

        sellable = true,
        price = 25
    }
}
```

### Step 2: Register with Items Component

```lua
-- server/main.lua

AddEventHandler('Core:Shared:Ready', function()
    for itemName, itemData in pairs(Config.Items) do
        COMPONENTS.Items:Register(itemName, itemData)
    end
end)
```

### Step 3: Handle Item Usage

```lua
-- server/items.lua

AddEventHandler('mythic-inventory:server:ItemUsed', function(source, characterId, itemName, slot, metadata)
    if itemName == 'energy_drink' then
        -- Restore energy
        COMPONENTS.Characters:UpdateCharacter(characterId, {
            ['metadata.needs.energy'] = 100
        })

        -- Give speed boost
        TriggerClientEvent('mythic-effects:client:SpeedBoost', source, 30000)

        -- Remove item
        COMPONENTS.Inventory:RemoveItem(characterId, slot, 1)

        -- Notify
        TriggerClientEvent('mythic-notifications:client:Send', source, {
            message = 'You feel energized!',
            type = 'success'
        })
    end
end)
```

### Step 4: Add Item Image

Place `energy_drink.png` (150x150px) in:
```
mythic-inventory/ui/assets/items/energy_drink.png
```

## Item Categories and Organization

Group related items for easy management:

```lua
-- Consumables
local Consumables = {
    ["water"] = { ... },
    ["sandwich"] = { ... },
    ["burger"] = { ... },
    ["cola"] = { ... }
}

-- Weapons
local Weapons = {
    ["weapon_pistol"] = { ... },
    ["weapon_smg"] = { ... },
    ["weapon_rifle"] = { ... }
}

-- Tools
local Tools = {
    ["lockpick"] = { ... },
    ["repairkit"] = { ... },
    ["fishing_rod"] = { ... }
}

-- Combine
Config.Items = {}
for k, v in pairs(Consumables) do Config.Items[k] = v end
for k, v in pairs(Weapons) do Config.Items[k] = v end
for k, v in pairs(Tools) do Config.Items[k] = v end
```

## Item Validation

Validate items before adding to inventory:

```lua
function ValidateItem(itemName)
    local item = COMPONENTS.Items:Get(itemName)

    if not item then
        return false, 'Item does not exist'
    end

    if not item.name or item.name == '' then
        return false, 'Invalid item name'
    end

    if not item.label or item.label == '' then
        return false, 'Invalid item label'
    end

    if not item.weight or item.weight < 0 then
        return false, 'Invalid item weight'
    end

    return true
end

-- Use in add item
function SafeAddItem(characterId, itemName, count)
    local valid, error = ValidateItem(itemName)

    if not valid then
        return false, error
    end

    return COMPONENTS.Inventory:AddItem(characterId, itemName, count)
end
```

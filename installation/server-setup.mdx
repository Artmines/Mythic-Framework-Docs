## Method 1: txAdmin Recipe Deployer (Recommended)

The **txAdmin Recipe Deployer** automatically installs Mythic Framework with all dependencies, database configuration, and proper resource loading order.

<Note>
**What is txAdmin?** txAdmin is the official FiveM server management panel. It comes pre-installed with FiveM server builds and provides web-based server management, recipe deployment, and more.
</Note>

### Prerequisites

Before using the recipe deployer, ensure you have:

<Steps>
  <Step title="FiveM Server">
    Download and install a FiveM server build (txAdmin is included)

    **Windows:** [Download from FiveM Artifacts](https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/)

    **Linux:** [Download from FiveM Artifacts](https://runtime.fivem.net/artifacts/fivem/build_proot_linux/master/)
  </Step>
  <Step title="Database Requirements">
    Install required databases on your server:
    - **MongoDB** (version 4.4 or higher) - For document storage
    - **MariaDB/MySQL** (version 10.5+ or MySQL 8.0+) - For relational data

    See [Database Setup](/installation/database-setup) for installation instructions
  </Step>
  <Step title="Node.js & pnpm">
    Install dependencies for UI resources:
    - **Node.js** (version 18.x or higher)
    - **pnpm** (package manager)

    ```bash
    # Install Node.js from https://nodejs.org/

    # Install pnpm globally
    npm install -g pnpm
    ```
  </Step>
</Steps>

### Recipe Deployment Steps

<Steps>
  <Step title="Start txAdmin">
    Launch your FiveM server with txAdmin for the first time:

    **Windows:**
    ```bash
    # Navigate to your FiveM server directory
    cd C:\FiveMServer

    # Start txAdmin
    FXServer.exe
    ```

    **Linux:**
    ```bash
    # Navigate to your FiveM server directory
    cd ~/fivem-server

    # Start txAdmin
    ./run.sh
    ```

    txAdmin will provide a setup URL (usually `http://localhost:40120`)
  </Step>

  <Step title="Access txAdmin Panel">
    Open your web browser and navigate to the txAdmin URL shown in the console

    **First-time Setup:**
    1. Create an admin account
    2. Link your FiveM license key (get one at [Cfx.re Keymaster](https://keymaster.fivem.net/))
    3. You'll be presented with deployment options
  </Step>

  <Step title="Deploy Mythic Framework Recipe">
    In the txAdmin deployment screen:

    1. Select **"Custom Recipe"** or **"Remote URL Template"**
    2. Enter the Mythic Framework recipe URL:
       ```
       https://raw.githubusercontent.com/Mythic-Framework/txAdminRecipe/main/mythic-stable.yaml
       ```
    3. Click **"Next"** or **"Deploy"**
  </Step>

  <Step title="Configure Deployment">
    The recipe deployer will prompt you for configuration:

    **Database Credentials:**
    - MongoDB connection string
    - MariaDB/MySQL host, port, username, password, database name

    **Server Settings:**
    - Server name
    - Max players
    - Server license key
    - Steam Web API key (optional)

    <Warning>
    **Important:** Have your database credentials ready before starting deployment. The recipe will test connections during setup.
    </Warning>
  </Step>

  <Step title="Wait for Deployment">
    The recipe deployer will:
    - Download Mythic Framework resources
    - Install npm dependencies for UI resources
    - Configure server.cfg and database connections
    - Set up resource loading order
    - Initialize database schemas

    **This may take 10-15 minutes** depending on your internet connection and server specs.

    <Tip>
    Watch the deployment log for any errors. Most issues are related to database connectivity or missing dependencies.
    </Tip>
  </Step>

  <Step title="Verify Installation">
    After deployment completes:

    1. Check the txAdmin dashboard shows "Server Online"
    2. Verify resources loaded successfully in the console
    3. Connect to your server via FiveM client
    4. You should see the Mythic Framework loading screen
  </Step>
</Steps>

### Post-Deployment Configuration

After successful deployment, you may want to customize:

<Accordion title="Server Settings" icon="gear">
  Edit `server.cfg` in txAdmin's file editor or directly:

  ```bash
  # Server identity
  sv_hostname "My Mythic Server"
  sv_maxclients 32

  # Server listing
  sets tags "mythic, roleplay, economy"
  sets banner_detail "https://url-to-your-banner.png"
  sets banner_connecting "https://url-to-your-banner.png"

  # OneSync (required for Mythic)
  set onesync on
  ```
</Accordion>

<Accordion title="Database Connections" icon="database">
  Database connections are auto-configured in:
  - `resources/[mythic]/mythic-base/config.lua` (MongoDB)
  - `resources/oxmysql/config.json` (MariaDB/MySQL)

  You can update these if your database credentials change.
</Accordion>

<Accordion title="Resource Order" icon="list-ol">
  Resource loading order is automatically set in `resources.cfg`.

  **Do not change the order** of core Mythic resources - they must load in a specific sequence.
</Accordion>

### Troubleshooting Recipe Deployment

<AccordionGroup>
  <Accordion title="Recipe URL not found" icon="link-slash">
    **Error:** "Could not fetch recipe from URL"

    **Solutions:**
    - Verify the URL is correct: `https://raw.githubusercontent.com/Mythic-Framework/txAdminRecipe/main/mythic-stable.yaml`
    - Check your server has internet access
    - Try accessing the URL in a web browser to verify it's accessible
  </Accordion>

  <Accordion title="Database connection failed" icon="database">
    **Error:** "Failed to connect to database"

    **Solutions:**
    - Verify database servers are running:
      ```bash
      # Check MongoDB
      mongosh --eval "db.version()"

      # Check MariaDB/MySQL
      mysql -u root -p -e "SELECT VERSION();"
      ```
    - Verify credentials are correct
    - Check firewall isn't blocking database ports (27017 for MongoDB, 3306 for MySQL)
    - Ensure databases exist and user has proper permissions
  </Accordion>

  <Accordion title="npm/pnpm errors during deployment" icon="npm">
    **Error:** "pnpm install failed" or "npm ERR!"

    **Solutions:**
    - Verify Node.js is installed: `node --version` (should be 18.x or higher)
    - Install pnpm globally: `npm install -g pnpm`
    - Check internet connection for package downloads
    - Manually run pnpm install in UI resource directories:
      ```bash
      cd resources/[mythic]/mythic-phone/ui
      pnpm install
      ```
  </Accordion>

  <Accordion title="Resources not loading" icon="box">
    **Error:** Resources show as "failed to load"

    **Solutions:**
    - Check resource directories exist in `resources/[mythic]/`
    - Verify `resources.cfg` was created properly
    - Check server console for specific error messages
    - Restart txAdmin completely
  </Accordion>
</AccordionGroup>

## Method 2: Manual Installation

For advanced users who want full control over the installation process.

### Download FiveM Server

<Tabs>
  <Tab title="Windows">
    1. Visit [FiveM Artifacts](https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/)
    2. Download the latest **recommended** build (green badge)
    3. Extract to your desired location (e.g., `C:\FiveMServer\`)
  </Tab>
  <Tab title="Linux">
    ```bash
    # Create server directory
    mkdir -p ~/fivem-server
    cd ~/fivem-server

    # Download latest recommended build (replace BUILD-NUMBER)
    wget https://runtime.fivem.net/artifacts/fivem/build_proot_linux/master/[BUILD-NUMBER]/fx.tar.xz

    # Extract
    tar xf fx.tar.xz

    # Make executable
    chmod +x run.sh FXServer
    ```
  </Tab>
</Tabs>

### Extract Mythic Framework

<Steps>
  <Step title="Obtain Mythic Framework">
    Extract the Mythic Framework package to a temporary location
  </Step>
  <Step title="Copy to Server Directory">
    Copy these folders/files into your FiveM server directory:
    - `resources/` folder
    - `configs/` folder
    - `server.cfg` file
    - `logo96.png` (optional - server icon)
  </Step>
  <Step title="Verify Structure">
    Your server directory should look like:
    ```
    FiveMServer/
    ├── FXServer.exe (or run.sh)
    ├── citizen/
    ├── configs/
    │   └── resources.cfg
    ├── resources/
    │   ├── [cfx-default]/
    │   ├── [mythic]/
    │   └── oxmysql/
    ├── server.cfg
    └── logo96.png
    ```
  </Step>
</Steps>

### Install Dependencies

You'll need to manually install dependencies for UI resources:

```bash
# Install Node.js from https://nodejs.org/ (version 18+)

# Install pnpm
npm install -g pnpm

# Install UI dependencies for each resource
cd resources/[mythic]/mythic-phone/ui
pnpm install

cd ../mythic-mdt/ui
pnpm install

cd ../mythic-hud/ui
pnpm install

# Repeat for other UI resources...
```

### Understanding Directory Structure

<Accordion title="resources/ Directory">
  Contains all server resources:

  ```
  resources/
  ├── [cfx-default]/     # Default FiveM resources
  ├── [mythic]/          # Mythic Framework (60+ resources)
  │   ├── mythic-base/   # MUST load first
  │   ├── mythic-pwnzor/ # Anti-cheat (load second)
  │   ├── mythic-characters/
  │   ├── mythic-inventory/
  │   └── ...
  └── oxmysql/           # MySQL connector
  ```

  <Warning>
  The `[mythic]` folder naming (with brackets) is important for organization in the server console.
  </Warning>
</Accordion>

<Accordion title="configs/ Directory">
  Contains resource loading configuration:

  ```
  configs/
  └── resources.cfg     # Resource load order
  ```

  **Load order is critical:**
  ```bash
  # Core CFX resources first
  ensure mapmanager
  ensure spawnmanager
  ensure sessionmanager

  # Database before Mythic
  ensure oxmysql

  # Mythic Framework
  ensure mythic-base     # MUST be first
  ensure mythic-pwnzor   # Anti-cheat second
  ensure mythic-characters
  # ... more resources
  ```
</Accordion>

### File Permissions (Linux)

```bash
# Set ownership
sudo chown -R youruser:youruser ~/fivem-server

# Make executables
chmod +x ~/fivem-server/run.sh
chmod +x ~/fivem-server/FXServer

# Set permissions
chmod 755 ~/fivem-server
```

## Mythic Framework Resources

Mythic Framework includes 60+ resources organized by function:

<AccordionGroup>
  <Accordion title="Core Resources (Load First)" icon="box">
    - `mythic-base` - Core framework, component system, database
    - `mythic-pwnzor` - Anti-cheat system
    - `mythic-queue` - Server queue and whitelist
    - `mythic-loadscreen` - Loading screen
  </Accordion>

  <Accordion title="Character & Player" icon="user">
    - `mythic-characters` - Multi-character system
    - `mythic-spawn` - Spawn management
    - `mythic-ped` - Character customization
  </Accordion>

  <Accordion title="Inventory & Economy" icon="box-open">
    - `mythic-inventory` - Full inventory system
    - `mythic-finance` - Banking and cash
    - `mythic-businesses` - Business ownership
    - `mythic-dealerships` - Vehicle dealerships
  </Accordion>

  <Accordion title="Jobs" icon="briefcase">
    - `mythic-jobs` - Job framework
    - `mythic-police`, `mythic-ems` - Emergency services
    - `mythic-mechanic` - Mechanic job
    - `mythic-restaurant` - Restaurant jobs
    - More job resources...
  </Accordion>

  <Accordion title="UI Systems" icon="desktop">
    - `mythic-hud` - Main HUD
    - `mythic-phone` - Smartphone system
    - `mythic-laptop` - Laptop interface
    - `mythic-mdt` - Mobile Data Terminal
    - `mythic-notifications` - Notifications
  </Accordion>
</AccordionGroup>

<Steps>
  <Step title="FiveM Server">
    ✅ FXServer.exe (Windows) or FXServer (Linux) exists

    ✅ server.cfg exists in root

    ✅ resources/oxmysql/ exists

    ✅ pnpm installed globally
    ✅ UI dependencies installed for each resource

## Comparison: Recipe vs Manual

| Feature | txAdmin Recipe | Manual Install |
|---------|---------------|----------------|
| **Setup Time** | 10-15 minutes | 1-2 hours |
| **Difficulty** | Easy | Advanced |
| **Database Config** | Automated | Manual |
| **Dependencies** | Auto-installed | Manual install |
| **Updates** | Via txAdmin | Manual download |
| **Best For** | New servers, quick setup | Experienced admins, custom setups |

<Tip>
**Recommendation:** Use the txAdmin Recipe unless you have specific reasons for manual installation. It's faster, easier, and less error-prone.
</Tip>

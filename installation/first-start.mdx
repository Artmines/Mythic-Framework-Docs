## Starting the Server

### Windows

<Tabs>
  <Tab title="Using Batch File">
    Create or use `start.bat` in your server root:

    ```batch start.bat
    @echo off
    title Mythic Framework Server
    echo Starting Mythic Framework...
    echo.

    FXServer.exe +exec server.cfg

    pause
    ```

    Double-click `start.bat` to start the server.
  </Tab>
  <Tab title="Command Line">
    ```bash
    # Open Command Prompt in server directory
    cd C:\Path\To\FiveMServer

    # Start server
    FXServer.exe +exec server.cfg
    ```
  </Tab>
</Tabs>

### Linux

```bash
# Navigate to server directory
cd ~/fivem-server

# Start server
./run.sh

# Or directly:
./FXServer +exec server.cfg
```

### Running as Background Service (Linux)

For production servers, run as a systemd service:

<Accordion title="Create systemd service">
```bash
# Create service file
sudo nano /etc/systemd/system/mythic-fivem.service
```

```ini mythic-fivem.service
[Unit]
Description=Mythic Framework FiveM Server
After=network.target mongodb.service mysql.service

[Service]
Type=simple
User=your_username
WorkingDirectory=/home/your_username/fivem-server
ExecStart=/home/your_username/fivem-server/FXServer +exec server.cfg
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# Reload systemd
sudo systemctl daemon-reload

# Enable service (start on boot)
sudo systemctl enable mythic-fivem

# Start service
sudo systemctl start mythic-fivem

# Check status
sudo systemctl status mythic-fivem

# View logs
sudo journalctl -u mythic-fivem -f
```
</Accordion>

### Expected Initialization

On first start, Mythic Framework will:

1. **Create database collections** (MongoDB)
   - `characters` collection
   - `inventory` collection
   - `users` collection
   - `logs` collection
   - And more...

2. **Create database tables** (MySQL)
   - Various persistent data tables
   - Initialized with defaults

3. **Initialize default data**
   - Default items for inventory
   - Default jobs
   - Default locations
   - System configurations

4. **Create indexes** for performance

All of this happens automatically - you don't need to do anything!


    **Solutions:**
    - Verify MongoDB and MySQL are running
    - Check connection strings in server.cfg

  </Accordion>

  <Accordion title="âŒ License Key Errors" icon="key">

    **Solutions:**
    - Verify license key is correct

### Verify Resource Status

In server console, type:


All Mythic resources should show as `started` in green.

## Connecting to Your Server

### From Same Machine (Localhost)

<Steps>
  <Step title="Open FiveM Client">
    Launch FiveM on the same computer as the server
  </Step>
  <Step title="Open Console">
    Press `F8` to open the console
  </Step>
  <Step title="Connect">
    ```
    connect localhost:30120
    ```
    (Use your port if different from 30120)
  </Step>
</Steps>

### From Different Machine (LAN/Internet)

<Steps>
  <Step title="Find Server IP">
    ```bash
    # Windows
    ipconfig
    # Look for IPv4 Address

    # Linux
    ip addr show
    # Look for inet address
    ```
  </Step>
  <Step title="Open Required Ports">
    Ensure port 30120 is open in:
    - Windows Firewall / UFW
    - Router port forwarding (for internet access)
  </Step>
  <Step title="Connect from FiveM">
    ```
    connect YOUR_IP:30120
    ```
  </Step>
</Steps>

## Post-Start Configuration

### Adjust Server Settings

Now that the server is running, you can fine-tune settings:

1. **Player count** - Adjust based on performance
2. **Resource configurations** - Each resource has config files
3. **Job settings** - Configure available jobs
4. **Economy** - Adjust starting money, prices

See [Configuration Guides](/configuration/server-cfg) for details.

### Add More Admins

Add other admin team members:

1. Have them connect to server
2. Run `status` in console to see their identifiers
3. Add to server.cfg:
   ```bash
   add_principal identifier.discord:THEIR_DISCORD_ID group.admin
   ```
4. Restart server

### Configure Discord Webhooks

If you haven't yet, set up Discord webhooks to monitor:
- Player connections
- Admin actions
- Errors and warnings
- Kill logs

See [Discord Webhooks Configuration](/configuration/discord-webhooks).

### Resource Usage Expectations

Typical resource usage for Mythic Framework:

| Resource | CPU (Idle) | CPU (48 players) | RAM |
|----------|------------|------------------|-----|
| mythic-base | 0.5-1% | 2-3% | 50MB |
| mythic-inventory | 0.1% | 1-2% | 30MB |
| mythic-hud | 0.3% | 0.5% | 20MB |
| mythic-phone | 0.1% | 0.3% | 15MB |
| All Mythic resources | 3-5% | 10-15% | 500MB |

<Note>
These are approximations. Actual usage varies based on player activity and server hardware.
</Note>

## Common First-Start Issues

<AccordionGroup>
  <Accordion title="Server crashes immediately" icon="circle-xmark">
    **Causes:**
    - Invalid license key
    - Database connection failure
    - Port already in use

    **Solutions:**
    - Check console for exact error
    - Verify prerequisites are running
    - See [Troubleshooting Guide](/installation/troubleshooting)
  </Accordion>

  <Accordion title="Can't connect to server" icon="wifi-slash">
    **Solutions:**
    - Verify server console shows "Server is ready"
    - Check firewall isn't blocking port
    - Try `connect 127.0.0.1:30120` if on same PC
    - Verify port forwarding if remote
  </Accordion>

  <Accordion title="Resources failing to load" icon="box-open">
    **Solutions:**
    - Check resources.cfg load order
    - Ensure mythic-base loads before other Mythic resources
    - Verify all resource folders exist in resources/[mythic]/
  </Accordion>

  <Accordion title="Character creation not appearing" icon="user-xmark">
    **Solutions:**
    - Check mythic-characters resource is started
    - F8 console - check for errors
    - Verify database connection
    - Try: `/logout` then reconnect
  </Accordion>

  <Accordion title="UI not showing (HUD, inventory, etc.)" icon="display-slash">
    **Solutions:**
    - Check mythic-hud, mythic-inventory are started
    - F8 console - check for JavaScript errors
    - Clear FiveM cache: Delete `%localappdata%\FiveM\FiveM.app\cache`
    - Restart FiveM client
  </Accordion>
</AccordionGroup>

## Server Maintenance Commands

Useful console commands:

```bash
# Gracefully restart server (gives players warning)
quit

# Restart a specific resource
restart mythic-inventory

# Reload server.cfg changes
refresh

# Kick a player
clientkick [player_id] "Reason"

# Show server info
sysinfo

# Show network stats
net_stats
```

## Backup Your Database

Now that your server is running, set up automatic backups:

<Tabs>
  <Tab title="MongoDB Backup">
    ```bash
    # Manual backup
    mongodump --db fivem --out /path/to/backup/$(date +%Y%m%d)
    mongodump --db auth --out /path/to/backup/$(date +%Y%m%d)

    # Automated daily backup (cron - Linux)
    0 4 * * * mongodump --db fivem --out /backups/$(date +\%Y\%m\%d)
    ```
  </Tab>
  <Tab title="MySQL Backup">
    ```bash
    # Manual backup
    mysqldump -u mythic -p MythicFramework_345AE9 > backup_$(date +%Y%m%d).sql

    # Automated daily backup (cron - Linux)
    0 4 * * * mysqldump -u mythic -pPASSWORD MythicFramework_345AE9 > /backups/mysql_$(date +\%Y\%m\%d).sql
    ```
  </Tab>
</Tabs>

<Warning>
**Critical:** Set up automated backups before going live with players. Database corruption or errors can result in data loss.
</Warning>
